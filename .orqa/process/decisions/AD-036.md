## Decision

Every artifact ID (matching patterns like `EPIC-NNN`, `TASK-NNN`, `AD-NNN`, `RES-NNN`, `RULE-NNN`, `PILLAR-NNN`, `MS-NNN`, `IDEA-NNN`, `IMPL-NNN`) that appears in markdown body text MUST be formatted as a proper artifact link: `[ID](ID)`.

This is enforced at two levels:

1. **Pre-commit hook** — A validation step in `.githooks/pre-commit` scans staged `.orqa/**/*.md` files for bare artifact IDs in body text (outside YAML frontmatter) and either auto-converts them to links or rejects the commit with a warning. IDs are validated against existing files before linking — only real artifact IDs are converted.
2. **Agent behaviour** — Agents and the orchestrator are expected to use linked references by default when writing artifact body text. The hook is a safety net, not the primary mechanism.

### Scope

- **Body text**: Bare IDs are converted to links
- **YAML frontmatter**: IDs in frontmatter fields (`id:`, `epic:`, `depends-on:`, `research-refs:`, `milestone:`, `pillars:`, etc.) are NOT converted — these are structured data references, not prose
- **Code blocks**: IDs inside fenced code blocks are NOT converted
- **Already-linked IDs**: IDs already in `[ID](ID)` format are skipped

## Rationale

Cross-linking is the connective tissue of the artifact graph. Every reference between artifacts creates a navigable relationship that helps users understand how work connects — which epics depend on which research, which tasks serve which epics, which decisions inform which rules.

Bare IDs (e.g. `[EPIC-a2fa3068](EPIC-a2fa3068)` without a link) are missed connections. They reference an artifact but don't create a navigable link in the rendered markdown. This makes the artifact graph incomplete and forces users to manually search for referenced artifacts.

By making cross-linking automatic:

- **The artifact graph is always complete** — every mention of an artifact creates a traversable link
- **No cognitive overhead** — authors don't need to remember the link syntax; write the ID and the hook fixes it
- **Consistency** — all artifacts follow the same linking convention
- **Discoverability** — clicking any artifact reference navigates to it in the app

## Consequences

- A new pre-commit hook step is needed in `.githooks/pre-commit` (or `.githooks/validate-schema.mjs`)
- The hook must parse YAML frontmatter boundaries to avoid modifying structured data
- The hook must skip fenced code blocks
- The hook must validate IDs against existing files to avoid creating dead links
- Agents should be updated to cross-link by default (reducing the hook's correction burden over time)
- The `orqa-documentation` skill should be updated to include cross-linking guidance
