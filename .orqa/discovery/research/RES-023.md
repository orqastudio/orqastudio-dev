## Problems Identified During Dogfooding

1. **Streaming broke on long responses** — Buffer overflow in NDJSON parser
2. **Tool output was unreadable** — Long file contents displayed inline without truncation
3. **Repeated tool calls cluttered the view** — 10 sequential Read calls each got their own card
4. **Sessions had no names** — All sessions showed as "New Session"
5. **Titlebar was generic** — Default OS titlebar didn't match the app's design language
6. **Error display was invisible** — Silent error swallowing in settings persistence

## Solutions

1. **Streaming fix** — Proper NDJSON line buffering with partial message handling
2. **Output truncation** — Tool outputs beyond 500 chars are collapsed with "Show more"
3. **Tool grouping** — Consecutive calls to the same tool are grouped into a summary card
4. **Auto-naming** — After first assistant response, session is auto-titled via LLM
5. **Custom titlebar** — Branded titlebar with window controls, project name, session indicator
6. **Error fix** — Silent error swallowing in settings persistence paths fixed
