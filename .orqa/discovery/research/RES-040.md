## Findings

### Layer Classification of Rules

44 rules audited. 8 rules marked `layer: core` contain project-specific content that violates portability:

| Rule | Universal Principle | Project-Specific Content |
|------|--------------------|-----------------------|
| [RULE-008](RULE-008) | Documentation-first | References Tauri/Svelte/Rust paths in body |
| [RULE-012](RULE-012) | Error ownership | Mentions cargo/clippy/npm |
| [RULE-018](RULE-018) | No aliases/hacks | Rust serde examples, TypeScript union examples |
| [RULE-020](RULE-020) | No stubs | Tauri invoke patterns, OrqaStudio-specific examples |
| [RULE-022](RULE-022) | Plan compliance | Systems Architecture Checklist lists Tauri commands, Svelte components |
| [RULE-025](RULE-025) | Root cleanliness | "What Belongs in Root" table lists Cargo.toml, svelte.config.js |
| [RULE-026](RULE-026) | Skill enforcement | Tier 2 injection table lists all orqa-* skills with file-pattern triggers |
| [RULE-031](RULE-031) | Vision alignment | "Foundational Principles" lists Tauri v2 + Svelte 5 + Rust + SQLite stack |

**Pattern:** Each rule correctly identifies a universal principle but embeds project-specific examples/checklists in the body. Fix: extract project content into project-layer companion rules or skills.

### Decisions Assessment

40 decisions. 4 supersession chains — all consistent (AD-005→[AD-032](AD-032), [AD-014](AD-014)→[AD-032](AD-032), [AD-015](AD-015)→[AD-021](AD-021), [AD-018](AD-018)→AD-019). Recent graph decisions (AD-038, [AD-039](AD-039), AD-040) are current and aligned.

**Issue:** Early decisions (AD-001 through AD-013) have no `layer` field. Later decisions (AD-033+) all have `layer: core`. Schema may not enforce `layer` on decisions.

### Lessons Assessment

15 lessons. 5 promoted (chains verified). 10 active — all still relevant. No stale or contradictory lessons found. [IMPL-007](IMPL-007) (don't refactor while delegating to agents) is a strong promotion candidate.

### Skills Assessment — Critical Issues

**[SKILL-046](SKILL-046) ID collision (CRITICAL):** Three skills share ID [SKILL-046](SKILL-046):
1. `plugin-setup` (plugin directory)
2. `plugin-setup` (symlinked to team/skills)
3. `research-methodology`

**SKILL-045 duplication without symlink:** `rule-enforcement` exists as separate copies in both `team/skills/` (layer: core) and `plugin/skills/` (layer: plugin). Content has diverged.

**Core-layer portability violations:**
- `composability` (SKILL-008): 37 project-specific references (Svelte components, sidecar paths, Tauri patterns)
- `orqa-native-search` (SKILL-013): Contains OrqaStudio-specific implementation details, should be `layer: project`
- `rule-enforcement` (team/skills copy): References OrqaStudio Rust enforcement engine paths, should be `layer: project`

**Layer/universality contradiction:** `orqa-code-search` (SKILL-005) is `layer: project` but [RULE-026](RULE-026) lists it as universal (every agent must have it).

### Agents Assessment

7 agents, all using `capabilities` field (aligned with RULE-040). [SKILL-046](SKILL-046) ID collision means agents referencing it get undefined behavior. The Planner and Researcher agents both reference [SKILL-046](SKILL-046).

### Documentation Assessment

- `orchestration.md` and `workflow.md` describe pre-graph-based patterns (ChunkHound-specific)
- No architecture doc exists for graph-based injection model (AD-038 accepted but no implementation doc)
- 15 wireframe/UI docs frozen at 2026-03-04 through 2026-03-07, likely outdated
- 3 rules have `scope: []` (empty) — [RULE-041](RULE-041), [RULE-042](RULE-042), [RULE-043](RULE-043)

### Plugin Assessment

Plugin hooks correctly implement graph-based injection. No remnants of old index-based approaches. The `rule-enforcement` skill duplication (not symlinked like `plugin-setup`) is the only issue.

### Rule Loading Tension

All 44 rules are loaded via `.claude/rules/` symlinks into every CLI session. This contradicts the "inject IDs only" principle — full rule bodies are injected regardless of relevance. This is the most significant architectural tension with the graph model.

## Recommendations

1. Fix [SKILL-046](SKILL-046) ID collision immediately
2. Fix SKILL-045 duplication — symlink or assign distinct IDs
3. Split 8 core rules: extract project-specific content into project-layer companions
4. Split composability skill: core principle + project examples
5. Fix skill layer misclassifications (orqa-native-search, rule-enforcement)
6. Update stale process docs (orchestration.md, workflow.md)
7. Create graph injection architecture doc
8. Address `.claude/rules/` full-body loading vs graph-based injection tension
