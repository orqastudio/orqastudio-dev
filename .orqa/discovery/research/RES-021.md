## Scope

A multi-day refactoring sprint (2026-03-06 to 2026-03-07) touching both backend
and frontend:

### Backend Extractions
- `stream_commands.rs` (2,425 lines) → 4 domain modules: `tool_executor`, `system_prompt`, `stream_loop`, `session_title`
- `setup_commands.rs` → `domain::setup`
- `governance_commands.rs` → `domain::governance_analysis`
- `artifact_commands.rs` → `domain::artifact_reader` + repo modules
- `stream_send_message` → focused helpers
- `project_settings` I/O → repo layer
- Timestamp utils → `domain::time_utils`
- `.orqa/` path constants → `domain::paths`
- Search commands → `OrqaError` instead of `String`

### Frontend Extractions
- `SettingsView` → focused sub-components
- `Toolbar` → focused sub-components
- Conversation store decoupled from session store
- `EmptyState` and `ErrorDisplay` → use `Button` component
- Missing error/loading states added across components

### Cross-Cutting
- 60+ hardcoded colors → semantic design tokens (warning/success variants)
- `make stop/restart` targets made cross-platform

## Pattern Established

This sprint formalized the domain service extraction pattern [AD-6cd1ff6f](AD-6cd1ff6f):
Command handler (thin) → Domain service (logic) → Repository (data access)
