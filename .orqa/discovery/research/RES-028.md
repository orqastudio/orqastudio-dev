## Problem Statement

We backfilled 33 historical artifacts (Wave 1), but the system that governs them hasn't caught up. Decisions exist as files in `.orqa/decisions/` but the framework doesn't define them. New decisions will still be added to the monolithic doc by habit. The roadmap doesn't reflect the new milestone. Agents don't know about `.orqa/decisions/`. Cross-references are unvalidated.

The artifact system must become self-sustaining: when someone creates a new decision, research doc, epic, or task, the rules and framework guide them to do it correctly without someone remembering to.

## What "Self-Sustaining" Means

1. **Every artifact type is defined** — schema, status workflow, storage location, creation rules
2. **Every artifact type is enforceable** — rules tell agents how to create them, lifecycle rules govern transitions
3. **Every artifact type is discoverable** — agents know where to look, CLAUDE.md lists the directory
4. **New work follows the patterns** — no one needs to remember; the rules make it the path of least resistance
5. **Cross-references are validated** — if an artifact references another, that reference resolves
6. **The monolithic doc has a clear role** — either it becomes an index pointing to individual artifacts, or it's retired

## Implementation: 5 Workstreams

### WS-1: Framework & Rules (makes the system self-sustaining)

These changes ensure that from this point forward, new artifacts are created correctly by default.

**1a. Add Decision type to `docs/product/artifact-framework.md`**

```yaml
# Decision (AD-NNN)
Storage: .orqa/decisions/AD-NNN.md
Schema:
  required: [id, title, status, created, updated, category]
  optional: [research-refs, supersedes, superseded-by, tags]
Status workflow: proposed → accepted → superseded | deprecated
```

**1b. Update `docs/process/artifact-workflow.md`**

Replace "record decisions in `docs/architecture/decisions.md`" with:
- Create `AD-NNN.md` in `.orqa/decisions/`
- Use the frontmatter schema from the framework
- Link back to the research that informed the decision via `research-refs`
- If a decision supersedes another, set `evolves-into` and update the old one's `evolves-from`

Add a "Decision Creation" section covering:
- When to create a decision (research produces an architectural choice)
- How to number it (next available AD-NNN)
- Required fields and links
- How supersession works

**1c. Update `.orqa/rules/artifact-lifecycle.md`**

Add Decision to the enforced artifact types:
- Status transitions: proposed → accepted → superseded | deprecated
- Creation rule: research that produces an architectural choice MUST create an AD-NNN.md
- Supersession rule: when a new decision replaces an old one, both must be updated
- Forbidden: adding decisions only to the monolithic doc without creating an individual artifact

**1d. Update `.claude/CLAUDE.md` resources table**

Add:
```
| Decisions | `.orqa/decisions/` | Architecture decision artifacts |
```

**1e. Update `.orqa/rules/architecture-decisions.md`**

Currently says "Read `@docs/architecture/decisions.md`". Update to also reference `.orqa/decisions/` as the source of individual decision artifacts.

### WS-2: Monolithic Doc Transition

**2a. Convert `docs/architecture/decisions.md` to an index**

The monolithic doc becomes a summary table that links to individual artifacts:

```markdown
| ID | Title | Status | Category | Artifact |
|----|-------|--------|----------|----------|
| AD-e513c9e4 | Thick Backend Architecture | accepted | architecture | [→ .orqa/decisions/AD-e513c9e4.md] |
| AD-a334623b | IPC Boundary Design | accepted | architecture | [→ .orqa/decisions/AD-a334623b.md] |
...
```

The full decision content lives only in the individual artifacts. The index provides discoverability and an overview.
