## Findings

### Layer Classification of Rules

44 rules audited. 8 rules marked `layer: core` contain project-specific content that violates portability:

| Rule | Universal Principle | Project-Specific Content |
|------|--------------------|-----------------------|
| [RULE-9daf29c0](RULE-9daf29c0) | Documentation-first | References Tauri/Svelte/Rust paths in body |
| [RULE-57ccb4a3](RULE-57ccb4a3) | Error ownership | Mentions cargo/clippy/npm |
| [RULE-9ba80a19](RULE-9ba80a19) | No aliases/hacks | Rust serde examples, TypeScript union examples |
| [RULE-e9c54567](RULE-e9c54567) | No stubs | Tauri invoke patterns, OrqaStudio-specific examples |
| [RULE-303c1cc8](RULE-303c1cc8) | Plan compliance | Systems Architecture Checklist lists Tauri commands, Svelte components |
| [RULE-1f30904a](RULE-1f30904a) | Root cleanliness | "What Belongs in Root" table lists Cargo.toml, svelte.config.js |
| [RULE-deab6ea7](RULE-deab6ea7) | Skill enforcement | Tier 2 injection table lists all orqa-* skills with file-pattern triggers |
| [RULE-1e8a1914](RULE-1e8a1914) | Vision alignment | "Foundational Principles" lists Tauri v2 + Svelte 5 + Rust + SQLite stack |

**Pattern:** Each rule correctly identifies a universal principle but embeds project-specific examples/checklists in the body. Fix: extract project content into project-layer companion rules or skills.

Recent graph decisions (AD-2783985c, [AD-e8ab8572](AD-e8ab8572), AD-6dfbba70) are current and aligned.

**Issue:** Early decisions (AD-e513c9e4 through AD-afc78f6e) have no `layer` field. Later decisions (AD-0291fa65+) all have `layer: core`. Schema may not enforce `layer` on decisions.

### Lessons Assessment

15 lessons. 5 promoted (chains verified). 10 active — all still relevant. No stale or contradictory lessons found. [IMPL-d5cbc9c7](IMPL-d5cbc9c7) (don't refactor while delegating to agents) is a strong promotion candidate.

### Skills Assessment — Critical Issues

**[KNOW-e3a559c9](KNOW-e3a559c9) ID collision (CRITICAL):** Three skills share ID [KNOW-e3a559c9](KNOW-e3a559c9):
1. `plugin-setup` (plugin directory)
2. `plugin-setup` (symlinked to team/skills)
3. `research-methodology`

**KNOW-bcfeb64e duplication without symlink:** `rule-enforcement` exists as separate copies in both `team/skills/` (layer: core) and `plugin/skills/` (layer: plugin). Content has diverged.

**Core-layer portability violations:**
- `composability` (KNOW-f0c40eaf): 37 project-specific references (Svelte components, sidecar paths, Tauri patterns)
- `orqa-native-search` (KNOW-40addb7a): Contains OrqaStudio-specific implementation details, should be `layer: project`
- `rule-enforcement` (team/skills copy): References OrqaStudio Rust enforcement engine paths, should be `layer: project`

**Layer/universality contradiction:** `orqa-code-search` (KNOW-1c5e752e) is `layer: project` but [RULE-deab6ea7](RULE-deab6ea7) lists it as universal (every agent must have it).

### Agents Assessment

7 agents, all using `capabilities` field (aligned with RULE-92dba0cb). [KNOW-e3a559c9](KNOW-e3a559c9) ID collision means agents referencing it get undefined behavior. The Planner and Researcher agents both reference [KNOW-e3a559c9](KNOW-e3a559c9).

### Documentation Assessment

- `orchestration.md` and `workflow.md` describe pre-graph-based patterns (ChunkHound-specific)
- No architecture doc exists for graph-based injection model (AD-2783985c accepted but no implementation doc)
- 15 wireframe/UI docs frozen at 2026-03-04 through 2026-03-07, likely outdated
- 3 rules have `scope: []` (empty) — [RULE-c95f4444](RULE-c95f4444), [RULE-f9d0279c](RULE-f9d0279c), [RULE-7f416d7d](RULE-7f416d7d)

### Plugin Assessment

Plugin hooks correctly implement graph-based injection. No remnants of old index-based approaches. The `rule-enforcement` skill duplication (not symlinked like `plugin-setup`) is the only issue.

### Rule Loading Tension

All 44 rules are loaded via `.claude/rules/` symlinks into every CLI session. This contradicts the "inject IDs only" principle — full rule bodies are injected regardless of relevance. This is the most significant architectural tension with the graph model.

## Recommendations

1. Fix [KNOW-e3a559c9](KNOW-e3a559c9) ID collision immediately
2. Fix KNOW-bcfeb64e duplication — symlink or assign distinct IDs
3. Split 8 core rules: extract project-specific content into project-layer companions
4. Split composability skill: core principle + project examples
5. Fix skill layer misclassifications (orqa-native-search, rule-enforcement)
6. Update stale process docs (orchestration.md, workflow.md)
7. Create graph injection architecture doc
8. Address `.claude/rules/` full-body loading vs graph-based injection tension
