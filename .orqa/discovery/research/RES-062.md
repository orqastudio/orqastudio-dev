## Research Questions

1. **Content audit**: For each doc — what does it cover, how long is it, is it focused or does it mix concerns?
2. **Duplication**: Which docs overlap with each other or with skills/rules?
3. **Graph connectivity**: How many docs have relationships? Which are orphaned? Which skills reference docs in body text without graph edges?
4. **Fitness for grounding**: Which docs (or sections) could serve as agent grounding if restructured?
5. **Restructuring plan**: What should the documentation tree look like? What needs splitting, merging, or rewriting?

## Methodology

- Read every file in .orqa/documentation/
- Count relationships in frontmatter
- Check which skills reference docs by path or ID in their body text
- Assess focus: does each doc serve one purpose or mix concerns?
- Identify grounding material: concise principle statements that could anchor agent behavior

## Findings

### Key Metrics

| Metric | Value |
|--------|-------|
| Total documentation files | 60 content files + READMEs |
| Total lines | ~20,000 |

| Stale phase references | 23 files |
| Missing pillar alignment | ~60% of files |

### Critical Issues

**1. Documentation is isolated from the graph.** 55% of docs have zero relationships. Agents can't traverse from a skill to its deeper documentation. Wireframes aren't linked to epics. UI specs aren't in designer Required Reading.

**2. Significant duplication exists.** 8 pairs of overlapping docs: architecture-overview duplicates core-architecture, governance-hub duplicates governance, coding-standards duplicates RULE-b49142be, guide/workflow duplicates process/workflow, component-inventory duplicates svelte-components.

**3. Stale content persists.** 23 files reference "Phase 2a/2b" numbering that no longer applies. launch-timeline.md is entirely outdated. Some docs are stubs (architecture-overview at 118 lines, metrics at 106 lines, priority-assessment at 92 lines).

**4. No grounding documents exist.** No doc is designed for agent injection as foundational purpose context. Vision.md is 400 lines — too long. Coding-standards is reference-only — no principles. Orchestration.md is 100% procedural — no purpose.

**5. Guide/ directory has unclear audience.** Three docs with no relationships, unclear whether they're for end users, agents, or developers.
