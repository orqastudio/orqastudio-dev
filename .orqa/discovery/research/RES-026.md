## Problem Statement

OrqaStudio is a clarity engine — users need to see what the AI receives (system prompt, context) and how it reasons (thinking output). The types, components, and store handling for AI transparency all exist and are production-ready. What's missing is the **emission logic** in the Rust backend that actually sends these events, and minor frontend wiring to ensure everything renders correctly end-to-end.

## Architectural Compliance

| Principle | How Satisfied |
|-----------|---------------|
| Error propagation | `SystemPromptSent` emission uses existing `Result` flow in `stream_commands.rs`. No new `unwrap()`. |
| IPC boundary | Events flow via existing `Channel<T>` — no new IPC mechanism. |
| Component purity | `ContextEntry.svelte` and `ThinkingBlock.svelte` receive props only. Store handles data. |
| Type safety | `StreamEvent::SystemPromptSent` already defined in Rust with `Serialize`. `StreamEventData` already defined in TypeScript. Types match. |
| Immutability | No new mutable state. `contextEntries` is accumulated via `$state` array push. |
| UX-first | Users see what was sent to the AI inline in conversation flow. Click to inspect details. |
| End-to-end completeness | All 4 layers exist: Rust event type + Channel emission + Svelte store accumulation + Svelte component rendering. This plan completes the Rust emission (layer 1) — the other 3 layers are already built. |
| Coding standards | Changes are <20 lines in `stream_commands.rs`. No new files needed. |

## Target UX

When a user sends a message, **before** the assistant's response appears, they see:

```
┌─ System prompt sent (4,231 chars) ─────── [click to inspect]
│
[Assistant]: Here's what I found...
```

Clicking the entry opens a dialog with:
- **Structured tab**: Sections showing Custom Prompt and Governance Prompt with char counts
- **Raw tab**: Full prompt text in a code block

During streaming, if thinking output arrives:
```
┌─ Reasoning... ────────────────────────── [expanded, live text]
│
[Assistant]: Here's what I found...
```

The thinking block auto-collapses when assistant text begins, and disappears after the turn completes (ephemeral).

## User Journeys

| Scenario | What Happens |
|----------|-------------|
| **Normal message** | User sends message → "System prompt sent (N chars)" appears inline → assistant response streams below |
| **Click to inspect** | User clicks context entry → dialog opens with Structured/Raw tabs showing full prompt content |
| **Thinking visible** | Thinking deltas arrive → amber block expands with live text → collapses when assistant text starts → disappears after turn |
| **No system prompt** | No project loaded → no `SystemPromptSent` emitted → no context entry shown → conversation works normally |
| **Error in prompt build** | `build_system_prompt()` fails → warning logged → no event emitted → conversation proceeds without transparency info |

## Component States

| Component | States |
|-----------|--------|
| `ContextEntry` | idle (shows summary text), hover (highlight), click (opens dialog) |
| `ContextDetailDialog` | open (structured tab default), tab switching (structured ↔ raw), closed |
| `ThinkingBlock` | streaming (expanded, live text, "Reasoning..." label), collapsed (text arrived), hidden (turn complete) |

## Phase 1: Backend Emission (single phase — this is a small epic)

### Task 1: Emit `SystemPromptSent` from `stream_commands.rs`

**Agent:** backend-engineer
**Scope:** `backend/src-tauri/src/commands/stream_commands.rs`

**What to do:**
In `stream_send_message()`, after `resolve_system_prompt()` succeeds and before `state.sidecar.send(&request)`, emit a `SystemPromptSent` event via `on_event`:

```rust
if let Some(ref prompt) = system_prompt {
    let _ = on_event.send(StreamEvent::SystemPromptSent {
        custom_prompt: None,  // EPIC-642234ba adds custom prompt support
        governance_prompt: prompt.clone(),
        total_chars: prompt.len() as i64,
    });
}
```

**Acceptance criteria:**
- `SystemPromptSent` event is emitted before the sidecar request is sent
- Event carries the full governance prompt text and correct char count
- `custom_prompt` is `None` ([EPIC-642234ba](EPIC-642234ba) will populate this)
- If no system prompt is resolved, no event is emitted (no error, no empty event)
- Emission failure (channel closed) does not prevent the message from being sent

### Task 2: Verify end-to-end rendering
