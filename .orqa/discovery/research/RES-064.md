## Research Questions

1. What should the dashboard answer at a glance?
2. What data is most valuable on first load vs. available on drill-down?
3. What dashboard patterns do similar tools (Linear, Notion, Jira, GitHub Projects) use well?
4. How should the dashboard differ for the three personas (Alex the Lead, Sam the Practitioner, Jordan the Independent)?
5. What layout structure best serves a clarity engine vs. a task tracker?

## Findings

### What the Dashboard Should NOT Be

- **Not a task/metric dump** — backlog is the Roadmap, KPIs are Metrics
- **Not a productivity signal** — should not encourage "more work" as success or measure velocity
- **Not a status board** — not optimised for executive scanning at 30,000 feet

### What the Dashboard SHOULD Be

The entry point to an ongoing conversation about the state of your work. Three questions mapping to three pillars:

1. **What is the current clarity level?** (Pillar 1) — Structural health of the artifact graph. Not "how many epics" but "how well-connected is my knowledge?"
2. **Is the system getting better?** (Pillar 2) — Trends across time. Not "99 errors found" but "errors were 120 last week, now 99 — improving"
3. **Are we staying true to purpose?** (Pillar 3) — Coherence between intention and action. Not "3 tasks in progress" but "these tasks serve your milestone, one is at risk of scope drift"

### Patterns from Similar Tools

| Tool | Pattern | Relevance | Adaptation |
|------|---------|-----------|------------|
| Linear | Issue health cards showing bottlenecks, not counts | Strong | Show pipeline congestion, not volume |
| Notion | Customisable database views with curated defaults | Strong | Customisable sections, opinionated defaults |
| Grafana | Status pages for different audiences | Strong | Different layouts per persona |
| Roam/Obsidian | Graph/vault stats, link density | Moderate | Graph health as narrative, not visualization |
| Coda | Integrated timeline + kanban + metrics | Strong | Roadmap integration on dashboard |

**Key insight**: Successful clarity tool dashboards show *coherence* and *trajectory*, not activity.

### Persona Needs

**Alex (The Lead)**: "What's the milestone status? Is governance healthy enough to delegate? What decisions need me?"
- Focus: Milestone status, graph health as "delegation readiness", decision queue

**Sam (The Practitioner)**: "Are standards being followed? What lessons should I apply? Quality trends?"
- Focus: Health trends, lesson promotion, scan pass rates, active rules

**Jordan (The Independent)**: "Can I safely delegate to agents? Is the project improving or accumulating debt?"
- Focus: Overall health score (simple), decision count (cognitive load), next actions

### Layout Options

#### Option A: Pillar-Based Three Columns
Three equal columns: Clarity | Learning | Purpose. Each contains its relevant widgets.
- **Pro**: Maps directly to vision
- **Con**: May feel abstract to new users

#### Option B: Narrative Flow (RECOMMENDED)
Top: Active milestone context. Below: Where you are | How you're improving | What's next.
- **Pro**: Reads as coherent story, natural left-to-right, milestone grounds everything
- **Con**: Assumes one active milestone

#### Option C: Persona-Adaptive
Detect persona and adapt layout. Alex sees strategic, Sam sees quality, Jordan sees simplified.
- **Pro**: Optimal per user
- **Con**: Implementation complexity

#### Option D: Smart Card Reordering
Current grid but cards reorder by urgency/attention needed.
- **Pro**: Minimal rework
- **Con**: Doesn't enforce narrative coherence, risks returning to metrics dump

### Recommended Sections

#### 1. Milestone Context (always first)
Active milestone title, gate question, P1 epic progress, deadline. If none: "No active milestone" + link to Roadmap. **Grounds everything in purpose.**

#### 2. Artifact Graph Health
Large status indicator (green/amber/red), summary score, breakdown: errors (broken refs, orphans), warnings (low connectivity). "Run integrity scan" action button. **Answers "can I trust my governance?"**

#### 3. Improvement Trends
2x2 sparkline grid: error count, warning count, rule count, pass rate. Current value + trend arrow + % change. Timeframe selector. **Answers "is the system getting better?"**

#### 4. Lesson Velocity
Single pipeline row: Draft → Review → Promoted → Active with counts. Summary text like "3 lessons awaiting promotion". **Shows governance is evolving.**

#### 5. Pending Decisions & Blockers
List of unanswered decisions blocking progress. Context, who decides, days pending. If empty: "No pending decisions — good to go". **Keeps decisions visible across sessions.**

#### 6. Knowledge Pipeline (optional, power users)
6-stage pipeline with connectivity health. Collapsible for Jordan, essential for Sam. **Shows governance maturity.**

### Recommendation

**Option B (Narrative Flow)** with these specifics:

1. **Top**: Active milestone card (always visible)
2. **Three columns**: Where you are (graph health) | Improving (trends + lessons) | What's next (decisions + blockers)
3. **Bottom** (collapsible): Knowledge pipeline + detailed breakdowns
4. **Mobile**: Stack vertically, keep milestone + three columns, hide bottom
5. **Persona micro-adaptations**: Alex emphasises delegation readiness, Sam emphasises quality trends, Jordan simplifies to overall health + next actions

### Implementation Phases

**Phase 1**: Milestone context card, refactored graph health widget, trend sparklines, decision queue placeholder
**Phase 2**: Lesson velocity, scope coherence indicator, orphan breakdown
**Phase 3**: Knowledge pipeline enhancement, persona-adaptive layouts, customisation UI

### Anti-Patterns to Avoid

- "Last 50 artifacts" list — activity ≠ clarity
- "Velocity this sprint" — measures activity, not learning
- Contribution heatmap — encourages activity gaming
- Generic KPI dashboard — misses OrqaStudio's identity
- Real-time "what changed this hour" — feeds context-switching
